1. 所有核心模块文件应直接放在项目根目录下
2. 避免使用 GenData 和 GenCode 等子目录存放核心模块
3. 数据文件和生成文件可以放在子目录中，但核心逻辑代码应保持在根目录

######三、数据结构设计
在数据类和全局变量中保存基本数据，数据结构简单清晰，必要时使用字典映射。
预定义的字符串类型，优化为枚举类型，不易出错而且可以获得IDE提示。
尽量不要在函数中定义新数据。
每个功能模块的代码需要做到数据和逻辑分离。
期望效果：数据驱动，灵活的数据结构和健壮的函数实现，最低成本修改逻辑处理代码。

# 4. 样式生成器（逻辑部分）
# -------------------------------------
class 样式生成器:
    @staticmethod
    def 创建样式(配置: 样式配置项) -> NamedStyle:
        """将样式配置转换为openpyxl的NamedStyle"""
        style = NamedStyle(name=配置.样式名称)
        style.font = Font(
            color=配置.字体.颜色.value,
            bold='bold' in 配置.字体.风格.value,
            italic='italic' in 配置.字体.风格.value,
            size=配置.字体.大小
        )
        if 配置.背景:
            style.fill = PatternFill(
                fill_type=配置.背景.填充类型,
                fgColor=配置.背景.颜色.value
            )
        if 配置.对齐方式:
            style.alignment = 配置.对齐方式
        return style

    @classmethod
    def 获取样式名称(cls, 表格类型: 表格类型, 样式类型: str) -> str:
        """获取指定表格类型的样式名称"""
        return 表格样式映射.get(表格类型, 表格样式映射[表格类型.默认样式])[样式类型]

    @classmethod
    def 获取样式配置(cls, 样式名称: str) -> 样式配置项:
        """根据名称获取样式配置"""
        return 样式预设库[样式名称]

    @classmethod
    def 清除样式缓存(cls, wb, style_name: str):
        target = style_name
        filtered = [
            style for style in wb._named_styles
            if target not in style.name
        ]
        # wb._named_styles是一个特殊的列表类型，必须转换回去
        wb._named_styles[:] = filtered


四、函数设计规范
1. 健壮性实现
2. 扩展性设计
在  参数可能改变、增减，甚至加入新类型参数的情况下，写健壮的处理函数；
下面是例子：
    def 注册表格样式(cls, wb, 表格类型: 表格类型 = 表格类型.默认样式):
        """根据表格类型注册对应样式"""
        # 获取样式名称
        header_style_name = 样式生成器.获取样式名称(表格类型, "header")
        data_style_name = 样式生成器.获取样式名称(表格类型, "data")

        # 强制清除旧样式
        样式生成器.清除样式缓存(wb, header_style_name)
        样式生成器.清除样式缓存(wb, data_style_name)

        # 获取新配置
        header_config = 样式生成器.获取样式配置(header_style_name)
        data_config = 样式生成器.获取样式配置(data_style_name)

        # 创建并注册新样式
        wb.add_named_style(样式生成器.创建样式(header_config))
        wb.add_named_style(样式生成器.创建样式(data_config))

        return header_style_name, data_style_name

    @classmethod
    def 生成表头(cls, ws, headers: list, 表头样式名称: str):
        for col_num, header in enumerate(headers, 1):
            cell = ws.cell(row=1, column=col_num, value=header)
            cell.style = 表头样式名称

    @classmethod
    def 更新或创建工作表(cls, 文件路径: str, 表名: str, 表格类型: 表格类型 = 表格类型.默认样式):
        """更新或创建工作表（修改参数名为表格类型）"""
        if os.path.exists(文件路径):
            wb = load_workbook(文件路径)
            表头样式名, 数据样式名 = cls.注册表格样式(wb, 表格类型)
            if 表名 in wb.sheetnames:
                del wb[表名]
            ws = wb.create_sheet(表名, 0)
        else:
            wb = Workbook()
            表头样式名, 数据样式名 = cls.注册表格样式(wb, 表格类型)
            ws = wb.active
            ws.title = 表名
        return wb, ws, 表头样式名, 数据样式名

五、防御性编程
可读性强，函数有清晰的功能注释，参数验证、逻辑处理等不同语句之间有明确的分界。
单模块中有清晰的注释划分区域，可读性更好，（比如：用#---------）。
使用语法糖简化代码
在函数内部做好参数的检查，及时发现因输入参数问题导致的bug。
极力避免隐式转换，尤其在函数内部，防止造成难以发现的bug。
在生成数据时，会标明生成数据的日期和相关的参数，方便对比数据，溯源数据信息

六、性能优化
在产生大量数据时，尽量用性能较高的方式处理，避免生成数据的时间过长

七、避免重复
发现代码中大量重复的部分。
减少逻辑处理上的重复，对相同的处理逻辑做进一步的抽象，更通用的抽象。
减少不方便的访问方式带来的重复，可以通过属性定义更便捷的访问方式。

八、注释和文档规范
1. 类注释
每个类都应该有清晰的文档字符串，说明其用途和功能：
```python
class 肉鸽技能数据类:
    """肉鸽技能数据类

    - 技能名称
    - 属性倍率（生命值、攻击力、攻速等）
    - 持续时间
    - 是否永久存在
```

2. 方法注释
每个方法都应该有详细的文档字符串，包括：
- 方法的功能描述
- 参数说明（类型和用途）
- 返回值说明
- 可能的异常说明

5. 注释风格
- 使用中文注释，保持与代码命名风格一致
- 注释应该简洁明了，避免冗长
- 对于复杂的算法或逻辑，可以添加详细的解释
- 使用统一的注释分隔符（如 # -------------------------------------）
- 对于TODO或FIXME等特殊注释，使用统一的格式：
```python
# TODO: 需要添加对特殊攻击类型的处理
# FIXME: 防御力计算可能存在溢出问题
```

6. 文档维护
- 当修改代码时，同时更新相关的注释
- 定期检查注释的准确性和时效性
- 删除过时或无用的注释
- 保持注释与代码的同步更新
