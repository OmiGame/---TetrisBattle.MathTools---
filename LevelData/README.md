# 关卡数据生成器

## 功能概述
关卡数据生成器用于根据关卡体验配置限制生成关卡波次配置，确保生成的波次符合以下要求：
1. 所有波次加起来等于最大波次数
2. 相邻两个波次体验尽可能不相同
3. 困难、地狱和Boss波次尽量不相连
4. 关卡内所有波次生成的boss总数不能超过3个
5. 可能生成boss的波次(地狱和boss波)按照关卡体验配置限制中的boss可能出现波次来配置
6. 第一波必须是轻松波次

## 新增功能 ✨
- **数据完整性检查**: 自动检查关卡数据的完整性和有效性
- **容错处理机制**: 跳过数据不完整的关卡，而不是直接报错停止
- **批量生成优化**: 能够处理关卡设计表中所有数据完整的关卡
- **详细统计报告**: 提供生成成功/失败的统计信息

## 主要类和方法

### 关卡生成器类

#### 核心生成方法
- `生成随机关卡配置(关卡ID, 蓝色等级, 绿色等级, 紫色等级, 阵容强度, 难度, 适配地图)`: 生成完整的关卡配置
- `生成关卡波次配置(蓝色等级, 绿色等级, 紫色等级, 阵容强度, 难度, 适配地图)`: 根据难度和配置限制生成波次配置列表

#### 批量处理方法 🆕
- `生成关卡设计表数据()`: 批量生成所有数据完整关卡的配置，自动跳过不完整数据
- `获取数据完整的关卡列表()`: 获取关卡设计表中所有数据完整的关卡信息
- `检查关卡数据完整性(关卡)`: 检查单个关卡数据是否完整

#### 辅助调整方法
- `调整波次数量(波次体验数量, 最大波次数)`: 调整各种波次体验的数量，确保总和等于最大波次数
- `创建波次体验序列(波次体验数量, 最大波次数, boss可能出现波次)`: 创建符合要求的波次体验序列
- `调整关卡波次配置(波次配置列表, 难度)`: 调整已生成的波次配置，使其符合要求

## 使用方法

### 🆕 批量生成关卡设计表数据（推荐）
```python
from LevelData.LevelDataGenerator import 关卡生成器

# 方法1: 直接生成所有数据完整的关卡
关卡设计表数据列表 = 关卡生成器.生成关卡设计表数据()
print(f"成功生成 {len(关卡设计表数据列表)} 个关卡配置")

# 方法2: 先检查数据完整性，再生成
完整数据关卡列表 = 关卡生成器.获取数据完整的关卡列表()
关卡设计表数据列表 = 关卡生成器.生成关卡设计表数据()
```

### 生成单个关卡配置
```python
from LevelData.LevelDataGenerator import 关卡生成器
from LevelData.LevelDataType import 难度类型

# 生成一个普通难度的关卡配置
关卡配置 = 关卡生成器.生成随机关卡配置(
    关卡ID=1,
    蓝色等级=10,
    绿色等级=12,
    紫色等级=8,
    阵容强度=3.5,
    难度=难度类型.普通,
    适配地图="森林"  # 可选参数，默认为"森林"
)

# 打印关卡基础信息
print(f"我方城堡血量: {关卡配置.我方城堡血量}")
print(f"敌方城堡血量: {关卡配置.敌方城堡血量}")
print(f"生成延迟时间: {关卡配置.生成延迟时间}")
print(f"限时生成时长: {关卡配置.限时生成时长}")
print(f"波次数量: {len(关卡配置.波次配置)}")

# 打印每个波次的体验类型
for i, 波次 in enumerate(关卡配置.波次配置):
    print(f"波次{i+1}: {波次.波次体验.value}")
```

## 🆕 错误处理和容错机制

### 数据完整性检查项目
生成器会自动检查以下项目：
1. **必要字段存在性**: 关卡编号、关卡难度、各品质等级、阵容强度、适配地图
2. **字段有效性**: 字符串字段不为空，数值字段不为None
3. **难度类型有效性**: 关卡难度必须是预定义的有效值
4. **数值范围合理性**: 等级在1到角色等级上限之间，阵容强度大于0

### 容错处理
- 遇到数据不完整的关卡时，自动跳过并记录
- 提供详细的跳过原因和统计信息
- 不会因为个别关卡数据问题导致整个批量生成失败

### 输出信息
生成过程会输出以下信息：
- 成功生成的关卡数量和ID
- 跳过的关卡列表和原因
- 数据完整性统计
- 各难度关卡分布统计

## 配置限制说明
关卡体验配置限制定义在`LevelDataType.py`中，包含以下内容：
- 各种波次体验的数量范围
- boss可能出现波次的范围
- 我方城堡血量
- 敌方城堡血量

每种难度都有对应的配置限制，生成器会根据这些限制生成符合要求的波次配置。
